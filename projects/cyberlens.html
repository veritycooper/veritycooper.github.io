Interactive Cybersecurity Education

CyberLens

A digital science museum for cyber security — playful visual exhibits that help beginners explore hashing, phishing, WiFi risk, and more without dense jargon.

Solo prototype · 2026 Designed & built end-to-end Live on GitHub Pages HTML · CSS · JavaScript

Overview

Cyber security can feel overwhelming when it is taught through dense terminology. CyberLens makes core ideas tangible: adjust a password and watch strength change, edit one character and see a hash transform, or tap clues in a suspicious message to understand phishing patterns.

I built it to combine security, user experience, accessibility, and educational design — helping people feel more confident online through exploration rather than fear. The prototype runs entirely on GitHub Pages with vanilla HTML, CSS, and JavaScript.

Why I built it

Cyber security is often taught through dense terminology, alarm-heavy messaging, or course-style modules that assume prior knowledge. I wanted something different for everyday users — especially people with little technical background who still need practical online safety skills.

CyberLens is aimed at curious beginners: students, career changers, or anyone who has heard words like “phishing” and “encryption” but never had a calm space to explore what they actually mean. The goal is confidence through understanding, not fear through jargon.

Design process

The first version felt too much like a traditional course — linear sections, lots of reading upfront, and a tone that still sounded like a textbook. I reframed the project around a science-museum metaphor: short exhibits, hands-on panels first, and optional deeper reading for visitors who want more.

Each exhibit went through a similar loop: pick one core idea, build a simple interaction, write plain-language copy, then add Museum Notes and Level Up content only where extra explanation helped. I kept removing features that felt clever but distracting, favouring clarity over completeness.

Challenges & tradeoffs

  • Simplifying without misleading. Hashing and encryption demos are educational sketches, not production cryptography — I had to balance accuracy with approachability.
  • Static-site constraints. Everything runs in the browser on GitHub Pages, so there is no backend, account system, or saved progress — preferences stay local to each device.
  • Calm vs. urgent. Security tools often lean on fear; I chose softer visuals and curiosity-driven copy, which meant being careful not to underplay real risks.
  • Scope control. Seven exhibits is already a lot for a solo prototype; deep-dive pages share one template to keep the codebase maintainable.

Inclusive design

Accessibility & settings

CyberLens is meant to feel welcoming to different learners, not only people comfortable with fast motion and low-contrast UI. The settings panel adjusts the local page only — no account, no server, and no tracking.

  • Reduce decorative motion — slows or removes ambient orbit animations and other non-essential movement for visitors who prefer a calmer screen.
  • Increase contrast — strengthens text, borders, and focus states so exhibits remain readable in brighter rooms or for users who need sharper visual separation.
  • Plain-language labels — exhibit copy avoids acronyms where possible; when terms like HTTPS or VPN appear on deep-dive pages, they are spelled out inline.
  • Keyboard-friendly structure — exhibits use semantic headings, buttons, and form controls so interactions stay usable without relying on precise mouse work alone.

Technical notes

CyberLens is a static site with no build step, no package manager, and no backend. All interactivity is handled in the browser with vanilla JavaScript modules loaded per page.

Information architecture

  1. Museum lobby — index.html introduces the concept and hosts inline exhibit panels visitors can scroll through.
  2. Deep-dive pages — exhibit.html?topic=... expands one topic with a larger experiment, a watch list, Level Up notes, and Museum Note takeaways.
  3. Shared styling — base.css handles layout and typography; cyberlens.css carries the museum visual language.
  4. Exhibit logic — cyberlens.js powers lobby interactions; cyberlens-exhibit.js renders each deep-dive from a single data structure.

Deployment

  • Hosted on GitHub Pages from the main branch
  • No API keys, databases, or server-side rendering
  • Accessibility preferences stored with localStorage
  • Repository includes setup and deployment notes in the README

What I learned

Building CyberLens pushed me to explain security ideas in plain language without flattening them into something wrong. I got more comfortable with educational UX — pacing, progressive disclosure, and knowing when to stop adding features.

It also reinforced how much accessibility and visual tone matter in security education: the same facts feel very different when they are delivered through alarm graphics versus calm, curiosity-led design. Shipping a complete static prototype end-to-end was good practice in scope control and maintainable front-end structure.

Inspiration

CyberLens draws on interactive science museums — spaces where you learn by trying things, wandering at your own pace, and leaving with one memorable idea rather than a wall of text. I also took cues from explainable technology demos and beginner-friendly security writing that respects the reader’s time.

The underlying belief is simple: people are more likely to build safer habits when they understand why something matters, not when they are overwhelmed by jargon or fear.
