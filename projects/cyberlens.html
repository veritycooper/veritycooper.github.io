Interactive Cybersecurity Education

CyberLens

An interactive cyber security education project inspired by a digital science museum. CyberLens helps beginners explore encryption, hashing, phishing, passwords, public WiFi, digital footprints, and social engineering through visual, hands-on exhibits — not dense technical lectures.

Status: Live prototype Focus: Accessible security education Stack: HTML, CSS, JavaScript
Try CyberLens View on GitHub

What is CyberLens?

Cyber security can feel overwhelming or overly technical for people with little background. CyberLens was built to make core ideas feel approachable: users explore one concept at a time through visuals, simple interactions, and real-world examples.

Rather than working like a traditional course, it encourages curiosity. You can adjust a password and watch strength change, edit one character and see a hash transform, or tap clues in a suspicious message to understand phishing patterns.

Why I built it

I wanted a project that combined cyber security, user experience, accessibility, and educational design. The goal was explainability and exploration — helping people feel more confident online without leaning on fear or jargon.

The full interactive prototype runs as a static site on GitHub Pages, with no build step or backend required.

Interactive exhibits

  • Password Playground — length, patterns, and strength feedback
  • Hashing Sandbox — see how a tiny text change transforms a fingerprint
  • Encryption Visualiser — a gentle lock-and-key style demo
  • Phishing Spotter — tap clues and compare safer patterns
  • Public WiFi Simulator — open vs protected traffic on a café network
  • Digital Footprint Explorer — how everyday data signals connect
  • Social Engineering Scenarios — practise safer responses

Technical notes

  • Built with vanilla HTML, CSS, and JavaScript — no framework
  • Main lobby and inline exhibits on index.html
  • Deeper “go further” exhibit pages via exhibit.html
  • Accessibility options include reduced motion and higher contrast
  • Source code and deployment docs live in the dedicated repository
